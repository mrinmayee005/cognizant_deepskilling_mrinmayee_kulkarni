Debug and Test with Chrome DevTools

DevTools Debugging Practice

Open Chrome DevTools

Press F12 or Ctrl+Shift+I (Windows) / Cmd+Option+I (Mac) to open DevTools.

Chrome DevTools Tips

Inspect Elements

Right-click any element and select "Inspect" to see its HTML and CSS. The Styles panel shows all applied rules.

Right-click > Inspect

Toggle Classes

In the Elements panel, click the + button in the Classes section to add/remove CSS classes dynamically.

.element.classList.toggle('active')

Box Model Visualizer

Click any element in the Elements tab to see its box model in the Computed tab. Hover over each part to highlight margins, borders, and padding.

Elements > Computed

Device Toolbar

Toggle the device toolbar to preview your page on different screen sizes. Press Ctrl+Shift+M.

Ctrl+Shift+M

Console Logging

Open the Console tab to see JavaScript output. Check for errors, warnings, and log messages from your scripts.

console.log('Debug message')

Network Tab

Monitor network requests to see if CSS files, images, and scripts are loading correctly. Look for 404 errors.

F12 > Network

Common CSS Errors to Debug

Error Type Symptom DevTools Fix
Missing semicolon Property below doesn't apply Check Styles panel for red error icon
Wrong selector Styles not targeting element Check specificity in Computed tab
Typo in property Property shows strikethrough Invalid properties are struck through in Styles
Missing closing brace All styles below break Check Console for parse errors
Incorrect units Layout looks wrong Edit values live in Styles panel to test
Z-index conflict Element hidden behind another Check stacking context in Elements panel
